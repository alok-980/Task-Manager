🏷️ Attributes vs Properties — Live Demo

An attribute is defined in HTML and reflects the initial/default value. A property is the live DOM value that changes as the user interacts. They start equal but diverge once the user types.

Property — input.value

This is the live DOM property. It reflects what the user currently has typed. Changes in real-time.

Attribute — getAttribute("value")

This is the HTML attribute. It stays fixed to the initial HTML value unless you call setAttribute().

🧠 Browser Rendering Pipeline

How the browser transforms raw HTML + CSS into pixels on your screen

HTML Path
📄 HTML Source
Raw bytes downloaded from server
↓
🔍 Parsing
HTML Parser reads bytes sequentially
↓
🪙 Tokenization
Tags converted to tokens: StartTag, EndTag, Text, Comment…
↓
🌳 DOM Tree
Tokens assembled into a tree of Node objects
CSS Path
🎨 CSS Source
Stylesheets, <style> tags, inline styles
↓
🔍 CSS Parsing
CSS Parser processes selectors and rules
↓
🗂️ CSSOM Tree
CSS Object Model — mirrors DOM but for styles
↘    ↙
DOM Tree + CSSOM Tree are combined
↓
✨ Render Tree
Only visible nodes with computed styles.
display:none elements are excluded.
↓
📐 Layout (Reflow)
Calculates exact position and size of every element
↓
🖌️ Paint
Fills in pixels — colors, borders, shadows, text
↓
🧩 Compositing
GPU combines layers and displays final output

Parsing

The HTML parser reads bytes sequentially and builds a tree. It is error-tolerant — malformed HTML is auto-corrected by the browser's parser.

Tokenization

The HTML is broken into tokens: <StartTag>, <EndTag>, Text, Comment, DOCTYPE. Each token maps to a node.

DOM Tree

A tree of Node objects (Element, Text, Comment…) that JavaScript can read and modify via DOM APIs.

CSSOM Tree

CSS rules are parsed into a tree structure with computed styles. Inheritance and specificity are resolved here before the Render Tree is built.

Render Tree

Combines DOM + CSSOM. Only visible elements appear. display:none nodes are excluded; visibility:hidden nodes are still included.

Event Bubbling

Events fire on the target element first, then propagate up through ancestors to document. Default behaviour (useCapture = false).

Event Capturing

Events fire on the outermost ancestor first and travel down to the target. Enabled by passing true as 3rd arg to addEventListener.

Event Delegation

Attach one listener on a parent to handle events from many children. Uses event.target to identify the source. Efficient — works for dynamic elements too.
